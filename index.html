1. Instance Methods

Definition

An instance method is a method defined within a class that operates on a specific instance (object) of that class. Unlike static methods, instance methods are tied to an object and can access its instance variables. They define object behavior and are invoked using an object.

Characteristics

  • Non-static: Declared without the static keyword.
  • Object-specific: Requires an object (e.g., object.method()).
  • Access to instance variables: Can read/modify fields.
  • Return type: Can return a value or be void.
  • Use of this: Refers to the current object’s fields/methods.

Example Program

public class Student {
    private String name;
    private int age;

    public void setDetails(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void displayDetails() {
        System.out.println("Name: " + name + ", Age: " + age);
    }

    public static void main(String[] args) {
        Student student = new Student();
        student.setDetails("Alice", 20);
        student.displayDetails();
    }
}
Student Object
name: null
age: 0

Tracing the Program

  1. Object Creation: Student student = new Student();
    • Creates a new Student object.
    • Initializes: name = null, age = 0.
  2. Calling setDetails: student.setDetails("Alice", 20);
    • Invokes setDetails.
    • this.name = name; sets name to "Alice".
    • this.age = age; sets age to 20.
  3. Calling displayDetails: student.displayDetails();
    • Prints: Name: Alice, Age: 20.
  4. Program Termination: The main method ends.

Output

Name: Alice, Age: 20

Key Points

  • Instance methods define object behavior.
  • this avoids parameter/field ambiguity.
  • Requires an object to call.

2. Constructors

Definition

A constructor is a special method that initializes objects, called automatically with new. It has the same name as the class and no return type.

Characteristics

  • Name: Matches the class name.
  • No return type: Not even void.
  • Automatic invocation: Called during object creation.
  • Overloadable: Multiple constructors with different parameters.
  • Default constructor: Provided by Java if none defined.

Types of Constructors

  • Default Constructor: Sets fields to defaults.
  • No-Argument Constructor: Custom initialization, no parameters.
  • Parameterized Constructor: Initializes with specific values.

Example Program

public class Book {
    private String title;
    private double price;

    public Book() {
        this.title = "Unknown";
        this.price = 0.0;
    }

    public Book(String title, double price) {
        this.title = title;
        this.price = price;
    }

    public void displayBook() {
        System.out.println("Title: " + title + ", Price: $" + price);
    }

    public static void main(String[] args) {
        Book book1 = new Book();
        book1.displayBook();
        Book book2 = new Book("Java Programming", 49.99);
        book2.displayBook();
    }
}
Book1 Object
title: null
price: 0
Book2 Object
title: null
price: 0

Tracing the Program

  1. Object Creation (book1): Book book1 = new Book();
    • Invokes no-argument constructor.
    • Sets title to "Unknown", price to 0.0.
  2. Calling displayBook for book1: Prints Title: Unknown, Price: $0.0.
  3. Object Creation (book2): Book book2 = new Book("Java Programming", 49.99);
    • Invokes parameterized constructor.
    • Sets title to "Java Programming", price to 49.99.
  4. Calling displayBook for book2: Prints Title: Java Programming, Price: $49.99.
  5. Program Termination.

Output

Title: Unknown, Price: $0.0 Title: Java Programming, Price: $49.99

Key Points

  • Constructors initialize objects.
  • Overloading allows flexible object creation.
  • No default constructor if any is defined.

3. Method Overloading

Definition

Method overloading allows multiple methods with the same name but different parameter lists, improving flexibility and readability.

Characteristics

  • Same method name.
  • Different signatures: Vary in number, type, or order of parameters.
  • Return type: Not considered for overloading.
  • Compile-time resolution: Compiler selects based on arguments.
  • Access modifiers: Can vary, but irrelevant to overloading.

Example Program

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }

    public int add(int a, int b, int c) {
        return a + b + c;
    }

    public double add(double a, double b) {
        return a + b;
    }

    public static void main(String[] args) {
        Calculator calc = new Calculator();
        System.out.println("Sum of 2 integers: " + calc.add(5, 10));
        System.out.println("Sum of 3 integers: " + calc.add(5, 10, 15));
        System.out.println("Sum of 2 doubles: " + calc.add(5.5, 10.5));
    }
}
Calculator Object
Result: 0

Tracing the Program

  1. Object Creation: Calculator calc = new Calculator();
  2. Calling add(int, int): calc.add(5, 10)
    • Selects add(int, int).
    • Returns 15.
    • Prints: Sum of 2 integers: 15.
  3. Calling add(int, int, int): calc.add(5, 10, 15)
    • Selects add(int, int, int).
    • Returns 30.
    • Prints: Sum of 3 integers: 30.
  4. Calling add(double, double): calc.add(5.5, 10.5)
    • Selects add(double, double).
    • Returns 16.0.
    • Prints: Sum of 2 doubles: 16.0.
  5. Program Termination.

Output

Sum of 2 integers: 15 Sum of 3 integers: 30 Sum of 2 doubles: 16.0

Key Points

  • Allows multiple methods with same name.
  • Compiler matches based on parameters.
  • Resolved at compile time.
